# Drone Frequencies & Modulation

This reference documents known drone communication frequencies and modulation types that can be detected by the Drone Detector.

## Detection Method

The Drone Detector identifies drones by matching two key characteristics:

1. **Frequency Range** - The specific radio frequency or frequency band
2. **Modulation Type** - How data is encoded in the signal (FSK, FHSS, LoRa, etc.)

When both characteristics match a known drone signature, the system identifies the drone type.

## Frequency Bands Overview

### Sub-GHz Bands (Detectable by T-Beam S3-Core)

| Band | Region | Common Uses |
|------|--------|-------------|
| 433 MHz | EU, Asia | Telemetry, remote control, some FPV |
| 868 MHz | EU | ISM band, telemetry, LoRa links |
| 915 MHz | US, AU | ISM band, long-range control, LoRa |

## Modulation Types

### LoRa (Long Range)

Proprietary spread-spectrum modulation by Semtech:

- **Characteristics**: Chirp spread spectrum, very long range
- **Bandwidth**: 7.8 - 500 kHz
- **Spreading Factor**: SF5 - SF12
- **Detection**: Use CAD (Channel Activity Detection)

```
Signature: Distinctive chirp pattern
Detection: RadioLib CAD function returns LORA_DETECTED
```

### FSK (Frequency Shift Keying)

Digital modulation shifting between two frequencies:

- **Characteristics**: Simple, robust, moderate range
- **Bit rates**: Typically 1.2 - 300 kbps
- **Detection**: Monitor RSSI for signal presence

```
Signature: Alternating frequency shifts
Detection: RSSI above noise floor with frequency deviation
```

### FHSS (Frequency Hopping Spread Spectrum)

Signal rapidly hops across multiple frequencies:

- **Characteristics**: Resistant to interference, harder to detect
- **Hop rate**: 10 - 1000+ hops/second
- **Detection**: Multiple brief detections across frequency range

```
Signature: Brief signals appearing at different frequencies
Detection: Pattern of short RSSI spikes across spectrum
```

### DSSS (Direct Sequence Spread Spectrum)

Signal spread across wide bandwidth:

- **Characteristics**: Low signal density, noise-like
- **Detection**: Requires correlation with known codes

```
Signature: Wide bandwidth, low power density
Detection: Challenging without known spreading code
```

### OOK (On-Off Keying)

Simplest modulation, carrier on or off:

- **Characteristics**: Simple remotes, short range
- **Detection**: RSSI pulses matching data pattern

```
Signature: Clear on/off pattern
Detection: Monitor for pulsed RSSI patterns
```

## Detection Algorithms

### Signature Matching

```cpp
struct DroneSignature {
  const char* name;
  float freqMin;
  float freqMax;
  ModulationType modulation;
  float bandwidth;
  int hopPattern;  // For FHSS
};

DroneSignature signatures[] = {
  {"ExpressLRS 915", 915.0, 930.0, MOD_LORA, 500.0, 0},
  {"Crossfire", 868.0, 870.0, MOD_LORA, 500.0, 0},
  {"DragonLink 433", 433.0, 435.0, MOD_FSK, 0, 0},
  {"SiK Telemetry", 915.0, 928.0, MOD_FSK, 0, 0},
  // Add more signatures...
};

DroneType matchSignature(float freq, ModulationType mod, float bw) {
  for (auto& sig : signatures) {
    if (freq >= sig.freqMin && freq <= sig.freqMax &&
        mod == sig.modulation) {
      return sig.name;
    }
  }
  return "Unknown";
}
```

### FHSS Detection

Detect frequency hopping by tracking multiple short detections:

```cpp
struct HopDetection {
  float frequency;
  unsigned long timestamp;
};

std::vector<HopDetection> recentDetections;

bool detectFHSS() {
  // Look for multiple detections across frequency range
  // within a short time window
  int hopCount = 0;
  unsigned long window = 1000; // 1 second
  
  for (auto& det : recentDetections) {
    if (millis() - det.timestamp < window) {
      hopCount++;
    }
  }
  
  // FHSS typically shows many brief detections
  return hopCount > 10;
}
```

## Limitations

1. **2.4 GHz / 5.8 GHz** - Most consumer drones use these bands which require different hardware
2. **Encrypted signals** - Cannot decode encrypted communications
3. **Spread spectrum** - DSSS signals may be below noise floor
4. **Fast FHSS** - Very fast hoppers may be missed between scans

## Expanding the Database

The signature database can be expanded by:

1. **Field capture** - Record signals from known drones
2. **Documentation** - Research published specifications
3. **Community** - Share signatures with other users
